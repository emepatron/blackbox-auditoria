# Site — Auditar estrutura e conteúdo (1.2 e 1.3)

O institucional precisa que o visitante ache o que quer e tenha motivo para confiar. Navegação resolve o primeiro; conteúdo e credibilidade resolvem o segundo.

## 1.2 Estrutura e navegação (cruze com a planilha)
1. **Menu claro + arquitetura de páginas essenciais.** Menu enxuto cobrindo Home / Sobre / Serviços / Contato / Blog (as que fizerem sentido). Cada página essencial existe e é alcançável.
2. **Rodapé com dados da empresa.** CNPJ + contato + links institucionais (privacidade, redes).
3. **Ausência de links quebrados.** Varra os links internos/externos — zero 404. Use um verificador de links ou cheque os principais à mão.
4. **Busca / breadcrumb** quando o site é extenso (bom ter; N/A em site pequeno).

## 1.3 Conteúdo e credibilidade (cruze com a planilha)
5. **Proposta de valor clara na home.** Em < 5s o visitante entende o que a empresa faz e para quem. Sem jargão vazio.
6. **Páginas de serviço/produto bem descritas.** Cada serviço com descrição real + CTA — não só um título.
7. **Prova social + "sobre" + clareza da empresa.** Depoimentos, cases, logos, números; página "sobre" que dá contexto; mensagem consistente entre páginas.
8. **Blog ativo e atualizado** (bom ter). Publicações recentes, sem lacunas longas. Blog abandonado passa impressão de empresa parada.

## Como auditar
- Navegue o site inteiro como um visitante primeiro (acho o que quero? confio?), como auditor depois (cada item acima).
- É institucional: avalie presença e credibilidade — não conversão de oferta única (isso é landing page) nem catálogo (isso é e-commerce).

## Saída
Status de cada item de 1.2 e 1.3 + observação específica (qual página, qual ajuste), para a skill `site-preencher`.
