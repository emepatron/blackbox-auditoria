01 · Manifesto

Princípios de design

Cinco regras que tornam um material reconhecível como V4 — antes mesmo de aparecer o logo.

01

Branco é a base

Fundo branco ou off-white. Espaço em branco gera autoridade. Não polua com cores ou gradientes desnecessários.

02

Vermelho é cirúrgico

O #FF0037 é detalhe, não decoração. Use em accent, label, número de seção, border-left, gráfico-chave. Nunca como fundo principal.

03

Números são protagonistas

Em diagnósticos, KPIs e dashboards: o número grande, em Space Grotesk Bold, é o herói. O texto explica, mas o número convence.

04

Tipografia faz o tom

Space Grotesk traz personalidade nos títulos. Inter dá legibilidade no corpo. JetBrains Mono adiciona autoridade técnica em labels e dados.

05

Bordas, não sombras

Cards e blocos usam borda fina 1px rgba(0,0,0,.10). Sombras pesadas envelhecem o material. Borda + radius leve = limpo e moderno.

06 · Disciplina

Anti-padrões — o que NÃO fazer

Esses são os erros mais comuns que descaracterizam a marca V4. Se você se pegar fazendo qualquer um desses, pare e revise.

❌ Não faça
  • Fundo vermelho ocupando o slide inteiro
  • Mais de 2 cores de destaque por slide
  • Sombras pesadas em cards (drop-shadow exagerado)
  • Gradientes coloridos como fundo principal
  • Fontes serifadas (Times, Georgia) em qualquer lugar
  • Comic Sans, Papyrus ou qualquer fonte decorativa
  • Emojis aleatórios decorando títulos
  • Bullets com bolinhas (•) — usar setas (→)
  • Bordas grossas (≥ 2px) em cards
  • Cores fora da paleta (verde-limão, rosa-bebê, etc.)
✅ Faça
  • Fundo branco ou off-white sempre
  • Vermelho V4 como detalhe (label, border-left, números-chave)
  • Borda fina 1px + radius leve em cards
  • Verde, laranja ou azul só com função (success/warning/info)
  • Space Grotesk em títulos · Inter no corpo · Mono em dados
  • Números grandes em Space Grotesk Bold
  • Setas → em bullets de ações/recomendações
  • Section number em mono (01 · TÍTULO)
  • Espaço em branco generoso entre seções
  • Letter-spacing positivo (0.1–0.2em) em labels uppercase
07 · Workflow

Como aplicar em cada ferramenta

Passo a passo prático para reproduzir o sistema em PowerPoint, Google Slides, Canva, Figma ou direto em HTML.

🅿PowerPoint

DESKTOP · MICROSOFT 365
  1. Instale as fontes localmente: Space Grotesk, Inter, JetBrains Mono (todas grátis no Google Fonts → baixar TTF → clique-duplo → instalar).
  2. Configure as cores em Design → Variantes → Cores → Personalizar Cores com os hex desta paleta.
  3. Use slide master (Exibir → Slide Mestre) com background #FFFFFF e top nav padrão.
  4. Para cada componente, use os specs em pt (PowerPoint usa pontos, não pixels).
  5. Salve o template como .potx para reutilizar.

🅖Google Slides

WEB · GOOGLE WORKSPACE
  1. As 3 fontes (Space Grotesk, Inter, JetBrains Mono) já estão disponíveis nativamente — basta selecionar em Fonte → Mais fontes e adicionar.
  2. Defina o tema em Slide → Editar tema: cor de destaque #FF0037, fundo #FFFFFF.
  3. Importe um pptx pronto da V4 e ajuste — todos os componentes deste guia funcionam em Slides.
  4. Use os specs em pt (Slides usa a mesma medida do PowerPoint).
  5. Para tabelas, use Inserir → Tabela e estilize manualmente conforme spec.

🎨Canva

WEB · APRESENTAÇÕES E POSTS
  1. Crie design com tamanho 1920×1080px (apresentação 16:9) ou 1080×1080px (post quadrado).
  2. Em Marca → Kit de marca, adicione as cores hex desta paleta e faça upload das fontes (Pro) ou use fontes equivalentes do Canva.
  3. Para cada componente, use os specs em px.
  4. Para o efeito border-left vermelho: crie um retângulo de 4px de largura colado à esquerda do card.
  5. Salve como template no kit da marca para reuso.

FFigma

WEB / DESKTOP · DESIGN PROFISSIONAL
  1. Crie um arquivo com Local Variables contendo todos os tokens de cor, tipografia e espaçamento desta paleta.
  2. Crie Components reutilizáveis para KPI Card, Bullet Card, Alert, Table Row, Badge.
  3. Use Auto Layout em todos os cards (padding 18-22px, gap 8-16px).
  4. Publique como Library para o time externo usar.
  5. Para exportar para PowerPoint: use plugin Pitch ou Slidedeck.

{ }HTML / CSS

WEB · APRESENTAÇÕES INTERATIVAS
  1. Importe as 3 fontes via Google Fonts no <head>.
  2. Cole o bloco :root deste guia (tokens de cor, fonte e espaçamento) no seu CSS.
  3. Copie os componentes diretamente do código-fonte deste arquivo (botão direito → Inspecionar).
  4. Para apresentações interativas: use Reveal.js ou monte slides como section.
  5. Para gerar PDF: use Chrome → Imprimir → Salvar como PDF.

📐Adobe Illustrator / Photoshop

DESKTOP · IMAGENS DE ALTA QUALIDADE
  1. Crie o arquivo em RGB (web) ou CMYK (impressão — converta o vermelho com cuidado, pode mudar a saturação).
  2. Configure Swatches com os hex da paleta. Salve como .ase e compartilhe.
  3. Use os specs em px (Photoshop) ou pt (Illustrator).
  4. Sempre exporte em PNG-24 (web) ou PDF/X (impressão).
